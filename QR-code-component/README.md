## 🛠️ My Process

### Built with

* **Semantic HTML5:** Used `<header>`, `<main>`, and `<footer>` for better structure.
* **CSS Custom Properties (Variables):** Defined colors and border-radius in `:root` for consistency.
* **Flexbox:** Used in the `body` to perfectly center the card on any screen size.
* **CSS Grid:** Applied to `.the-container` to organize the layout rows.
* **Google Fonts:** Integrated the 'Outfit' font family.

### 💡 What I Learned (Mistakes & Corrections)

This project wasn't just about writing code; it was about learning professional standards:

1.  **Avoid Fixed Heights:** I initially used a fixed `height`, but I learned that it's a mistake in responsive design. I now use `padding` and `auto` height to let elements breathe.
2.  **Proper CSS Ordering:** I learned to order properties logically: Dimensions (Width) -> Layout (Display) -> Spacing (Padding/Margin) -> Aesthetics (Colors/Fonts).
3.  **The Centering Rule:** I memorized that `margin: auto` only works on `block` elements with a defined `width`.
4.  **Responsive Best Practices:** I learned why `max-width` is better than a fixed `width` to ensure the card fits smaller mobile screens.
