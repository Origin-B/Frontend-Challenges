# Testimonials Grid Section

![Project Screenshot for Mobile](./design/mobile-preview.png)
![Project Screenshot for Desktop](./design/desktop-preview.png)

A responsive testimonials grid section built with pure HTML & CSS. This project is a solution to the [Testimonials Grid Section challenge](https://www.frontendmentor.io/challenges/testimonials-grid-section-Nnw6J7Un7) on Frontend Mentor.

---

## 📌 Overview

This project displays user testimonials in a clean, responsive grid layout. The design shifts from a single-column mobile view to a multi-column desktop grid using CSS Grid.

### The challenge
- Build a testimonials grid that looks identical to the design
- Create a responsive layout that works on all screen sizes
- Use semantic HTML and well-organized CSS

---

## 🛠️ Built With

- **Semantic HTML5** — Using `main`, `section`, `article`, `header` for meaningful structure
- **CSS Custom Properties** — Centralized color variables in `:root`
- **CSS Grid** — For desktop layout (4-column grid)
- **CSS Flexbox** — For aligning content inside each card
- **Mobile-first workflow** — Starting from mobile then enhancing for desktop
- **BEM Methodology** — Organized and reusable class names

---

## 📱 Responsive Design

| Screen Size | Layout |
|-------------|--------|
| **Mobile** (≤ 1023px) | Single column, cards stacked vertically |
| **Desktop** (≥ 1024px) | 4-column grid with cards spanning multiple columns |


---

## 🎯 What I Learned

- **CSS Grid placement** — Using `grid-column` and `grid-row` to position cards
- **BEM naming** — Writing clean, maintainable class names like `.card`, `.card--purple`, `.card__title`
- **Background image positioning** — Placing the quotation mark icon in the purple card
- **Responsive design** — Using `min-width` media queries to switch from mobile to desktop
