🚀 Project: Order Summary Component
📝 Overview
This is a solution to the Order Summary Component challenge on Frontend Mentor. This project was a major turning point in my 100 Days of Code journey, moving from basic layouts to mastering modern CSS alignment.

💡 What’s New? (The Leap from Project 1 to Project 2)
Comparing this to my first project (the Recipe Page), here is the technical evolution I achieved:

1. From Normal Flow to Flex-box Mastery
Project 1 (Recipe): Relied on the standard block model where elements are stacked. Alignment was done via margin: auto and max-width.

Project 2 (Order Summary): I fully embraced Flex-box. I used display: flex with flex-flow: row wrap and justify-content: space-evenly to create a dynamic and organized layout.

2. Advanced CSS Variables with Fallbacks
Project 1: Used basic variables for colors.

Project 2: Implemented Fallback Values (e.g., var(--Blue-50, hsl(225, 100%, 98%))). This ensures the design remains intact even if a variable fails, a true "Owner" mindset for robust code.

3. Interactive UI/UX States
Project 1: Had basic link hovers.

Project 2: Added sophisticated :hover effects on buttons and links, including transitions for font-weight, background-color, and box-shadow to improve the user experience.

4. Semantic HTML5 Architecture
I evolved from simple sections to a more complex structure using main, article, and nested section tags, making the code more readable and SEO-friendly.
